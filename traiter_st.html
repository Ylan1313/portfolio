Projet BUT 2 — Traiter

Modélisation robuste de données wafer

Développement d’un script Python qui automatise le nettoyage des données, la sélection des variables corrélées, l’entraînement de modèles polynomiaux régularisés et la recherche d’un minimum théorique.

Langage Python
Données BIN & _AVERAGE
Modèles Ridge / Lasso / ElasticNet
Sortie CSV + outil interactif
← Retour à la compétence Traiter

Objectif du projet

L’objectif était de construire un pipeline automatisé capable d’analyser plusieurs variables cibles de type BIN et de trouver les paramètres de procédé les plus influents.

Problématique

Comment identifier automatiquement les variables les plus explicatives, comparer plusieurs modèles et obtenir une équation exploitable sans refaire toute l’analyse à la main ?

Données utilisées

Le projet repose sur un fichier de données industrielles contenant des variables liées à la production de wafers. Les colonnes de type BIN représentent les variables cibles à analyser, tandis que les colonnes se terminant par _AVERAGE correspondent aux paramètres explicatifs utilisés pour construire les modèles.

Variables cibles

Les colonnes BIN sont utilisées comme variables à expliquer. Le programme analyse chaque BIN séparément afin d’identifier les paramètres qui peuvent influencer son comportement.

Variables explicatives

Les colonnes _AVERAGE représentent les mesures moyennes disponibles dans le jeu de données. Elles servent à rechercher les corrélations et à construire les modèles prédictifs.

Contrôle qualité

Avant la modélisation, le script vérifie la cohérence des lignes, notamment en contrôlant que la somme des colonnes BIN reste proche de 100.

Méthodologie

Le projet reprend une logique de pipeline complet : préparation des données, traitement des valeurs aberrantes, sélection de variables, entraînement de modèles et export des résultats.

Préparation des données

Lecture du fichier CSV, conversion des colonnes numériques, sélection des colonnes BIN et _AVERAGE, puis vérification de la cohérence des lignes.

Gestion des valeurs aberrantes

Le script propose plusieurs traitements robustes : winsorisation par quantiles, clipping par MAD ou suppression des lignes extrêmes avec l’IQR.

Sélection et modélisation

Pour chaque cible, les variables les plus corrélées sont sélectionnées avec Spearman ou Pearson, puis testées dans des modèles Ridge, Lasso et ElasticNet.

Validation et interprétation

Les modèles sont comparés avec validation croisée, R² et MAE. Les meilleurs résultats sont exportés avec leurs équations et coefficients.

Fonctionnement détaillé

Le script a été conçu pour limiter les manipulations manuelles. Il enchaîne automatiquement les étapes de préparation, de sélection, d’entraînement et d’export des résultats.

Filtrage des colonnes

Le programme repère automatiquement les colonnes BIN et _AVERAGE, puis conserve uniquement les variables exploitables pour l’analyse.

Traitement des valeurs extrêmes

Plusieurs méthodes de gestion des outliers sont proposées : clipping par quantiles, méthode MAD ou suppression des lignes extrêmes avec l’IQR.

Recherche du meilleur modèle

Pour chaque cible, le script teste plusieurs modèles régularisés avec différents degrés polynomiaux, puis classe les résultats selon leurs performances.

Export des résultats

Les modèles retenus sont exportés dans un fichier CSV contenant les métriques, les variables utilisées, les coefficients et les équations exploitables.

Simulation de l’outil

Cette simulation garde l’esprit de l’interface interactive du projet : choix de la cible, choix de la méthode, sélection automatique des variables, génération d’un modèle et test des paramètres avec des curseurs.

Difficultés rencontrées

Une des principales difficultés a été de construire un pipeline robuste capable de fonctionner sur plusieurs variables cibles sans devoir adapter le code manuellement à chaque analyse.

Il a aussi fallu gérer les valeurs aberrantes, éviter la fuite de données entre l’entraînement et le test, puis comparer les modèles de manière fiable.

Solutions apportées

Pour répondre à ces problèmes, le script sépare clairement les données d’entraînement et de test, applique une validation croisée répétée et conserve uniquement les modèles ayant des performances suffisantes.

L’ajout d’une interface interactive permet ensuite de tester rapidement différentes valeurs de paramètres et de visualiser leur effet sur la prédiction.

Résultat du projet

Le programme permet de parcourir automatiquement les variables cibles, de comparer plusieurs modèles et d’exporter un fichier de synthèse prêt à être utilisé pour l’analyse ou la visualisation.

Modèles classés

Les meilleurs modèles sont conservés par cible avec leurs métriques : R² CV, MAE CV, R² train et R² test.

Équations exploitables

Le script produit des équations standardisées et dénormalisées afin de faciliter l’interprétation métier.

Décision facilitée

L’outil interactif aide à tester des valeurs de paramètres et à estimer un gain potentiel.

Cette approche rend l’analyse plus rapide, plus reproductible et plus fiable qu’une exploration manuelle, tout en gardant une présentation claire pour un portfolio.
