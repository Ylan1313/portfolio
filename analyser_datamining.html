Projet BUT 3 — Analyser

Data Mining : prédiction des catégories de véhicules

Projet de classification supervisée réalisé à partir du Forza Final Dataset. L’objectif était de prédire la catégorie stock_specs d’un véhicule à partir de ses caractéristiques techniques et qualitatives.

Dataset Forza Final Dataset
Données 525 lignes
Variables 22 colonnes
Cible stock_specs
← Retour à la compétence Analyser
Objectif du projet

Classer automatiquement les véhicules

Le projet avait pour but de prédire automatiquement la catégorie d’un véhicule : S2, S1, A, B, C ou D. Cette catégorie correspond à la variable cible stock_specs.

La prédiction repose sur des caractéristiques comme la vitesse, l’accélération, la tenue de route, la transmission, le type de véhicule ou encore certains éléments liés à ses performances globales.

L’enjeu principal était de comparer plusieurs modèles de classification et de comprendre l’impact du choix des variables sur la qualité de prédiction.

Projet de classification

Objectif analytique

Le projet vise à comprendre quelles caractéristiques influencent le plus la catégorie d’un véhicule et à construire un modèle capable de généraliser correctement sur de nouvelles données.

6 catégories prédites
6 modèles comparés
4 corpus étudiés

L’analyse compare plusieurs approches de data mining afin d’identifier le meilleur compromis entre performance, stabilité et interprétation.

Méthodologie

Prétraitement des données

La variable Stock_Rating a été retirée pour éviter une fuite d’information. Certaines variables numériques comme la vitesse, l’accélération et les g-force ont été converties pour être correctement exploitées.

Encodage des variables catégorielles

Les variables qualitatives ont été transformées afin d’être utilisées par les modèles. Cette étape permet d’intégrer les informations non numériques comme le type de véhicule ou la transmission.

Séparation train / test

Les données ont été séparées en deux parties : 80 % pour l’apprentissage et 20 % pour le test. Cette séparation permet d’évaluer la capacité des modèles à généraliser sur de nouvelles données.

Comparaison des modèles

Six modèles ont été comparés : ZeroR, KNN, SVM, Decision Tree, Random Forest et XGBoost. Les performances ont été évaluées avec l’accuracy, la precision, le recall et le F1-score.

Résultats globaux des modèles

Les meilleurs résultats sont obtenus par SVM, Decision Tree et XGBoost. Le Decision Tree se distingue par son F1-score, tandis que SVM obtient le meilleur recall.

Modèle Accuracy Precision Recall F1-score
ZeroR 21% 3% 17% 6%
KNN 46% 32% 47% 35%
SVM 79% 80% 82% 78%
Decision Tree 78% 83% 81% 79%
Random Forest 75% 73% 75% 72%
XGBoost 79% 75% 80% 76%

Meilleur compromis

Decision Tree

Modèle retenu comme particulièrement intéressant grâce à son équilibre entre performance, lisibilité et interprétation des décisions de classement.

Pourquoi ce choix ?

Performance solide 78% accuracy
Meilleur F1-score 79%
Très bonne précision 83%

Même si le SVM obtient aussi de très bons résultats, le Decision Tree est plus simple à expliquer visuellement. Il permet donc de mieux justifier les décisions du modèle dans un contexte de portfolio.

Lecture des différents modèles

SVM

Très performant globalement, avec une accuracy de 79 % et le meilleur recall. Il identifie bien les catégories, mais son interprétation est moins directe.

Decision Tree

Très bon compromis entre performance et lisibilité. Son F1-score de 79 % permet de le considérer comme l’un des modèles les plus pertinents du projet.

XGBoost

Modèle performant avec 79 % d’accuracy. Il reste cependant plus complexe à expliquer qu’un arbre de décision simple dans le cadre d’un portfolio.

Random Forest

Résultats corrects avec 75 % d’accuracy, mais légèrement inférieurs aux meilleurs modèles. Il reste utile pour comparer avec une approche ensembliste.

KNN / ZeroR

KNN reste limité sur ce jeu de données, tandis que ZeroR sert surtout de modèle de référence. Leurs résultats montrent l’intérêt des modèles plus avancés.

Répartition et construction des corpus

Les variables ont été regroupées en plusieurs corpus pour analyser l’impact du choix des données. Cette comparaison permet de voir si le modèle a besoin de toutes les informations ou si certains groupes de variables suffisent.

Répartition train / test

80% Train
20% Test

Le jeu de données a été séparé en deux parties : une partie majoritaire pour entraîner les modèles et une partie test pour évaluer leurs performances sur des données non vues.

Corpus 1

Global

Toutes les variables sont utilisées, sauf Stock_Rating, supprimée pour éviter une fuite d’information.

Corpus 2

Performance only

Corpus centré sur les variables de performance : speed, handling, acceleration, launch et offroad.

Corpus 3

Filtré numérique

Corpus numérique où certaines variables ont été retirées : Top_Speed, 0-60_Mph, 0-100_Mph et g-force.

Corpus 4

Qualitatif

Corpus composé uniquement de variables qualitatives comme Model_type, Drive_Type et groupe_poids.

Impact du choix des corpus

Résultats selon les corpus

Global 78% F1 : 79,5%
Performance only 68% F1 : 68%
Filtré numérique 70% F1 : 70,7%
Qualitatif 52% F1 : 55%

Le corpus global donne les meilleurs résultats. Cela montre que la combinaison des variables numériques et qualitatives apporte plus d’information que l’utilisation d’un groupe de variables isolé.

Difficultés rencontrées

Identifier la fuite d’information

Certaines variables pouvaient donner trop directement la réponse au modèle. La variable Stock_Rating a donc été supprimée pour obtenir une évaluation plus réaliste.

Construire les corpus

Il fallait choisir quels groupes de variables comparer. Cette étape a demandé de réfléchir au sens des variables et à leur rôle dans la prédiction.

Comparer plusieurs métriques

Un modèle ne peut pas être jugé uniquement avec l’accuracy. La precision, le recall et le F1-score ont permis d’avoir une vision plus complète.

Interpréter les résultats

Certains modèles sont performants mais plus difficiles à expliquer. Le Decision Tree était intéressant car il proposait un bon compromis entre performance et lisibilité.

Bilan du projet

Ce projet m’a permis de comprendre l’importance du prétraitement et du choix des variables dans une tâche de classification. Les résultats montrent que les meilleurs modèles ne sont pas seulement ceux qui obtiennent le meilleur score, mais aussi ceux dont les résultats peuvent être interprétés correctement.

Le SVM, le Decision Tree et XGBoost obtiennent les meilleurs résultats globaux. Cependant, le Decision Tree ressort comme un modèle particulièrement intéressant grâce à son équilibre entre performance, adaptabilité aux données restreintes et lisibilité du classement.
